# Estado System - Correct Structure

## PEDIDOS (5 states only)

**DINÁMICOS** (auto-recalculate on stock changes):
- Bajo pedido - At least 1 product in backorder
- Combinado - Mix: some in stock, some in backorder
- Completo - All products in stock

**SEMIFIJO** (fixed by inventory, reverts to dynamic after finanzas):
- Listo para remisionar - Inventory confirmed goods prepared (CAN REVERT)

**FINAL** (immutable):
- Surtido completo - Fully invoiced, never changes

## PRODUCTOS / detallesdelpedido.estado_producto (4 states only)

**DINÁMICO** (auto-recalculate on stock changes):
- Bajo pedido - Insufficient stock
- Con stock - Stock available

**SEMIFIJO** (fixed by inventory, can revert after finanzas):
- Surtido - Inventory confirmed product prepared (CAN REVERT)

**FINAL** (immutable):
- Facturado - Finanzas confirmed & invoiced

## CRITICAL RULE: CXC Generation

**CXC is created ONLY when estado_producto = 'Facturado' (finanzas confirms), NOT before**

- MULTIPLE CXCs can exist per pedido (one per product finanzas confirmed)
- CXC is generated by trigger when product status changes to 'Facturado'
- Not generated manually in controllers
- es_credito flag on pedido determines if CXC is created
